Claude Code Notifications on Android TV

Wire up a Claude Code Stop hook that pops a notification on your Android TV with a Hebrew summary of Claude's last reply — visible over any content.

End result: Every time Claude Code finishes a response in your terminal, a notification appears on your TV with the project name and a one-sentence summary of the reply. Works over Netflix, YouTube, anything playing.

Prerequisites

  • A Google TV / Android TV (tested on TCL).
  • A Mac with Claude Code installed (verify: claude --version).
  • Mac and TV on the same Wi-Fi network.
  • jq and curl on the Mac (both shipped by default on macOS).

1 Install TvOverlay on the TV

TvOverlay is a free (with paid tier) app that exposes an HTTP API on the TV and displays overlay notifications above any content.

What to look for

  • Open Google Play Store on the TV and search for TvOverlay.
  • Package: com.tabdeveloper.tvoverlay
  • If it doesn't appear in on-TV search, open this URL in a browser on your Mac/phone: https://play.google.com/store/apps/details?id=com.tabdeveloper.tvoverlay and hit Install selecting your TV as the target device.

Figure 1 — TvOverlay on the Google Play Store

2 Get the TV's IP address

Open the TvOverlay app on the TV. On the main screen, look at the bottom-left: you'll see a QR code labelled TvOverlay Remote, and directly below it the line "You can also connect manually" — that's where the IP address and port appear. The port is usually 5001.

Figure 2 — TvOverlay — the IP lives under the QR code, in the "You can also connect manually" section (circled in red)

Heads up: The IP may change after a reboot of the TV or router. If notifications stop arriving, re-check the IP and update the script. Consider reserving a static DHCP lease for your TV in the router.

For the rest of this guide we'll use 192.168.1.149:5001 as a placeholder — replace with your actual address.

3 Smoke test — send a manual curl

Before wiring the hook, confirm the basic path works. From your Mac terminal:

curl -s -X POST http://192.168.1.149:5001/notify \
  -H "Content-Type: application/json" \
  -d '{"title":"Test","message":"Hello from the Mac","duration":8,"corner":"top_end"}'

If a popup appears on the TV, you're good to go. If not, verify the TV is on, TvOverlay is running, and it has the "Display over other apps" permission granted.

4 Create the hook script

Create the directory ~/.claude/hooks and the file tv-notify.sh:

mkdir -p ~/.claude/hooks
nano ~/.claude/hooks/tv-notify.sh

Paste the following (change the IP at the top to match yours):

#!/bin/bash
# Claude Code Stop hook → TvOverlay notification on Android TV.

if [[ -n "$TV_NOTIFY_RUNNING" ]]; then
  exit 0
fi

TV="http://192.168.1.149:5001/notify"

INPUT=$(cat)
CWD=$(printf '%s' "$INPUT" | jq -r '.cwd // empty')
TRANSCRIPT=$(printf '%s' "$INPUT" | jq -r '.transcript_path // empty')
PROJECT=$(basename "${CWD:-$PWD}")

RAW=""
if [[ -n "$TRANSCRIPT" && -f "$TRANSCRIPT" ]]; then
  RAW=$(jq -r '
      select(.type=="assistant" and .message.role=="assistant")
      | (.message.content
         | if type=="array" then map(select(.type=="text").text) | join(" ") else . end)
      | select(. != null)
      | gsub("\\s+"; " ")
      | gsub("^\\s+|\\s+$"; "")
      | select(length > 0)
      ' "$TRANSCRIPT" 2>/dev/null \
    | tail -1 \
    | sed -E 's/```[^`]*```//g; s/`[^`]*`//g' \
    | sed -E 's/[[:space:]]+/ /g; s/^ //; s/ $//')
fi

BODY=""
if [[ -n "$RAW" ]] && command -v claude >/dev/null 2>&1; then
  PROMPT="The text below is an assistant's response from a coding session. Summarize what the assistant did or answered, in ONE short Hebrew sentence (up to 15 words). Return ONLY the Hebrew sentence.

BEGIN_TEXT
${RAW}
END_TEXT"
  BODY=$(TV_NOTIFY_RUNNING=1 perl -e 'alarm shift; exec @ARGV' 25 claude -p --model haiku "$PROMPT" 2>/dev/null \
    | tr '\n' ' ' | sed -E 's/[[:space:]]+/ /g; s/^ //; s/ $//')
fi

if [[ -z "$BODY" ]]; then
  BODY="${RAW:0:250}"
  [[ ${#RAW} -gt 250 ]] && BODY="${BODY}…"
fi
[[ -z "$BODY" ]] && BODY="(no text in last response)"

MESSAGE="📁 ${PROJECT}"$'\n\n'"${BODY}"

PAYLOAD=$(jq -cn \
  --arg title "🤖 Claude finished" \
  --arg msg "$MESSAGE" \
  --arg src "Claude Code" \
  '{title:$title, message:$msg, source:$src,
    smallIcon:"mdi:robot-happy", smallIconColor:"#CC785C",
    largeIcon:"mdi:check-circle",
    corner:"top_end", duration:15}')

curl -s -m 3 -X POST "$TV" \
  -H "Content-Type: application/json" \
  -d "$PAYLOAD" > /dev/null 2>&1 &

exit 0

Save (Ctrl+O, Enter, Ctrl+X in nano) and make the file executable:

chmod +x ~/.claude/hooks/tv-notify.sh

5 Register the hook in settings.json

Open ~/.claude/settings.json and add the hooks section:

{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "/Users/YOUR_USERNAME/.claude/hooks/tv-notify.sh"
          }
        ]
      }
    ]
  }
}
Important: Replace YOUR_USERNAME with your macOS username (check with echo $USER). You must use an absolute path here — ~ is not expanded by Claude Code.

6 Live test

  1. Open a new terminal window (settings are loaded at the start of each Claude Code session).
  2. cd into any project: cd ~/code/my-project
  3. Run: claude
  4. Ask for something short: "say hi"
  5. The moment Claude finishes replying → a notification pops on the TV with the project name and a Hebrew summary of the reply.

Customization

Colors and icons

  • smallIcon / largeIcon: any Material Design Icon as mdi:robot-happy, mdi:alert, mdi:bell, etc.
  • smallIconColor: hex, e.g. #CC785C (Claude orange), #FF0000 (red), #00FF00 (green).

Position and duration

  • corner: top_start, top_end, bottom_start, bottom_end. Note that start / end flip under RTL system locale — if the TV is in Hebrew, top_start maps to the right-hand corner.
  • duration: display time in seconds (8-15 feels right).

Skip the haiku summary (free and instant)

If you'd rather not spend on haiku calls, delete the whole if [[ -n "$RAW" ]] block in the script. The fallback (raw text, truncated to 250 chars) will run instead — zero latency, zero cost, but no Hebrew translation.

Troubleshooting

  • No notifications appearing: Check the IP is correct, TvOverlay is running with the "Display over other apps" permission, and the Mac/TV are on the same network.
  • Summary comes in English: haiku occasionally ignores the instruction. If it persists, re-word the PROMPT in the script or try --model sonnet.
  • Hook feels slow: our tests ran 8-18 seconds. Claude Code allows Stop hooks up to 60 seconds before timing out.
  • Temporarily disable: chmod -x ~/.claude/hooks/tv-notify.sh. Re-enable with chmod +x.
