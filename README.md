# claude-tv-notify

Show notifications on your Android TV every time **Claude Code** finishes a reply — with the project name and a short Hebrew (or English) summary of what Claude just said, as an overlay that sits on top of whatever you're watching.

<p align="center">
  <img src="docs/images/IMG_1226.jpg" alt="TvOverlay on a TCL Google TV showing the pairing screen" width="650">
</p>

Requires a [Google TV / Android TV](https://www.android.com/tv/) with the free [**TvOverlay**](https://play.google.com/store/apps/details?id=com.tabdeveloper.tvoverlay) app installed (it exposes a local HTTP server that draws overlay popups) and a Mac running [Claude Code](https://claude.com/claude-code).

---

## Why this exists

Claude Code sessions often run for minutes at a time on long tasks — builds, refactors, test suites. Instead of keeping the terminal window in front of you, you get a notification on your TV the moment Claude is done, and you can glance up from whatever you're doing.

Android TV's built-in notifications are practically invisible (tiny toast, no notification drawer), so this uses **TvOverlay** — a purpose-built Play Store app that draws real overlay popups above any content, including Netflix, YouTube, and games.

## What you get

- Native popup on the TV every time Claude Code's `Stop` hook fires.
- Title: `🤖 Claude finished`, source: `Claude Code`.
- First body line: the project directory (`📁 my-repo`).
- Second body line: a one-sentence **Hebrew summary** of the last reply (generated by `claude -p --model haiku`). Easy to switch to English or any other language — see [Customization](#customization).
- Falls back to the raw last message (truncated) if the `claude` CLI isn't available or times out.
- Non-blocking: the notification fires in the background, so Claude's own UX isn't delayed even if the TV is offline.

## Quick start

### 1. Install TvOverlay on the TV

From the Google Play Store on your TV search for **TvOverlay** (package `com.tabdeveloper.tvoverlay`). If it doesn't show up in TV search, open this URL on your phone or laptop and send the install to your TV from there:

<https://play.google.com/store/apps/details?id=com.tabdeveloper.tvoverlay>

Open the app once and grant the **"Display over other apps"** permission when prompted.

### 2. Read the TV's IP

Open **TvOverlay** on the TV. Under the `TvOverlay Remote` QR code on the bottom-left you'll see a line labelled **"You can also connect manually"** — the IP and port live right there (the port is `5001` by default).

### 3. Install the hook on your Mac

```bash
# Prerequisites (both shipped with macOS, but confirm):
command -v jq >/dev/null || brew install jq
command -v claude >/dev/null && claude --version

# Put the hook in place
mkdir -p ~/.claude/hooks
curl -fsSL https://raw.githubusercontent.com/OzlevyQ/claude-tv-notify/main/hooks/tv-notify.sh \
  -o ~/.claude/hooks/tv-notify.sh
chmod +x ~/.claude/hooks/tv-notify.sh

# Point the hook at your TV (replace with your actual IP)
sed -i '' 's|YOUR_TV_IP|your-ip-address|' ~/.claude/hooks/tv-notify.sh
```

### 4. Register the hook in `~/.claude/settings.json`

Merge this into your existing file (keep your other top-level keys intact):

```json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "/Users/YOUR_USERNAME/.claude/hooks/tv-notify.sh"
          }
        ]
      }
    ]
  }
}
```

> The `command` **must** be an absolute path — Claude Code does not expand `~`.

### 5. Smoke test

First, confirm TvOverlay is reachable from your Mac:

```bash
curl -s -X POST http://your-ip-address:5001/notify \
  -H "Content-Type: application/json" \
  -d '{"title":"Hello","message":"Smoke test","duration":8,"corner":"top_end"}'
```

A popup should appear on the TV within ~1s. If it doesn't, confirm the IP is right, TvOverlay is running on the TV, and both devices are on the same Wi-Fi.

### 6. Use it live

Open a **new** terminal window (Claude Code reads `settings.json` at session start), `cd` into any project, run `claude`, and ask it something. The moment it finishes answering, a notification lands on the TV.

## How it works

Claude Code's `Stop` hook fires at the end of every assistant turn. It passes the hook script a JSON object on stdin containing, among other things, `cwd` and `transcript_path` (a JSONL file of every event in the session).

The script:

1. Reads `transcript_path` and extracts the text content of the **last** assistant message — just that one turn, not the whole session.
2. Sends the text to `claude -p --model haiku` with a prompt asking for a one-sentence Hebrew summary.
3. Posts a JSON payload to TvOverlay's `/notify` endpoint with the project name + summary.
4. Backgrounds the `curl` with a 3-second timeout, so an offline TV never blocks Claude's UI.

A recursion guard (`TV_NOTIFY_RUNNING` env var) prevents the inner `claude -p` call from re-triggering the Stop hook on itself.

The summarization prompt used, verbatim:

> The text below is an assistant's response from a coding session. Summarize what the assistant did or answered, in ONE short Hebrew sentence (up to 15 words). Return ONLY the Hebrew sentence, with no preamble, no quotes, no markdown.

## Customization

The notification payload is built at the bottom of `hooks/tv-notify.sh`. TvOverlay supports:

| Field | What it does |
|-------|--------------|
| `title` | Top line of the popup. |
| `message` | Body — the hook sets this to `📁 PROJECT` + blank line + summary. |
| `source` | Small label above the title. |
| `smallIcon` | Tiny icon, top-left. Accepts [Material Design Icons](https://pictogrammers.com/library/mdi/) as `mdi:<name>` (e.g. `mdi:robot-happy`). |
| `smallIconColor` | Hex, optionally with `#`. Default here is `#CC785C` (Claude orange). |
| `largeIcon` | Big icon next to the text. Also accepts MDI names, image URLs, or base64. |
| `image` | Full-width image (URL or base64). |
| `video` | RTSP/HLS/DASH/SmoothStreaming source — TvOverlay plays it in the popup. |
| `corner` | `top_start`, `top_end`, `bottom_start`, `bottom_end`. **Note:** `start`/`end` flip under RTL system locale. |
| `duration` | Seconds the popup stays on-screen. |

### English instead of Hebrew

Swap the Hebrew instruction in the `PROMPT` variable for e.g.:

```
Summarize what the assistant did or answered, in ONE short English sentence (up to 15 words).
```

### Skip the summarization step

If you don't want to spend ~0.1¢ per turn on haiku, delete the whole `if [[ -n "$RAW" ]] && command -v claude …` block. The fallback (raw last message, truncated to 250 chars) runs instead — instant and free, but no translation.

### Different models

`claude -p --model sonnet` produces noticeably better Hebrew phrasing if haiku's output feels off; the cost goes up and the call gets slower (roughly 15–25s vs 8–12s).

## Troubleshooting

| Symptom | Check |
|---------|-------|
| No popup at all | IP correct in `tv-notify.sh`? TV on? `curl -s http://IP:5001/` returns `200`? Same Wi-Fi? TvOverlay has "Display over other apps"? |
| Popup on TV but wrong side | TV locale is RTL — `top_end` becomes left. Use `top_start` for right-hand corner in Hebrew/Arabic locales. |
| Summary is in English | Haiku occasionally ignores the Hebrew instruction. Try rewording the `PROMPT` or switch to `--model sonnet`. |
| Hook feels slow | Expected 8–18s with haiku. Claude Code allows Stop hooks up to 60s before timing out. |
| Temporarily disable | `chmod -x ~/.claude/hooks/tv-notify.sh`. Re-enable with `chmod +x`. |
| IP changed after router reboot | Set a static DHCP lease for the TV in your router so the address doesn't move. |
